## 1. Regular Maintenance Tasks

### What should be checked regularly?

1. **Check all links monthly** - Ensure all navigation and internal links work correctly and fix broken ones  
2. **Update content quarterly** - Review information about marine life and update outdated facts  
3. **Test website on different browsers monthly** - Make sure the site still works properly on Chrome, Edge, etc.  
4. **Check image loading monthly** - Verify all images display correctly and are not broken  
5. **Review form submissions weekly** - Check if users sent messages through the form  
6. **Backup files monthly** - Keep a copy of all HTML and CSS files in case something breaks  

---

## 2. Updating Content

### Adding new content

1. Open the correct HTML file  
2. Copy an existing section or structure  
3. Replace text, images, and information  
4. Add new images to the image folder  
5. Test the page layout  
6. Save changes  
7. Push to GitHub 

### Updating existing content

1. Open the relevant HTML file  
2. Use the search function (Ctrl + F) to find the section you want to update  
3. Edit the text, images, or information  
4. If updating an image, replace the file in the image folder or update the file path  
5. Save the file  
6. Open the page in the browser to test if everything is working correctly  
7. Commit the changes using Git  
8. Push the changes to GitHub  

### Removing outdated content

1. Open the HTML file where the content is located  
2. Find the section you want to remove  
3. Delete the HTML code related to that content  
4. Remove any related images from the image folder if they are no longer needed  
5. Check if any links are pointing to that content and remove or update them  
6. Test the page to make sure nothing is broken  
7. Commit the changes  
8. Push the changes to GitHub

## 3. Technical Maintenance

## 6. Common Problems and Solutions

### Problem 1  
**Problem:** Image not showing  
**Possible causes:** Wrong path or file name  
**Solution:** Check file path and spelling  

### Problem 2  
**Problem:** Layout broken  
**Possible causes:** CSS conflict or missing class  
**Solution:** Check CSS rules and inspect element  

### Problem 3  
**Problem:** Links not working  
**Possible causes:** Wrong file name or path  
**Solution:** Fix link and test navigation

## 7. Version Control

### Git workflow

1. Pull latest version  
2. Make changes  
3. Test locally  
4. Commit with clear message  
5. Push to GitHub  
6. Check live version  

If something breaks:
- Revert to previous version  
- Fix problem  
- Commit again  


### Good commit messages

Good:
- "Fixed image alignment"
- "Updated marine life content"
- "Added new page"

Bad:
- "changes"
- "update"
